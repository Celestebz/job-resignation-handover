# 资产分类参考指南

## 三类资产分类法

离职时所有资产分三类，这是skill的核心判断框架。

## 🔴 公司资产（必须交接，不能带走）

### 判断标准
- 用公司时间/设备/账号产出
- 为公司业务服务
- 含商业敏感信息（客户、价格、合同）
- 公司付费的工具/账号

### 常见项目
| 项目 | 说明 |
|---|---|
| 客户数据 | 跟进表、询盘、订单、谈判进度 |
| 报价体系 | 报价单/PI模板、价格底线、折扣规则 |
| 合作方记录 | 合同、付款进度、合作条款、坑点备注 |
| 展会资料 | 客户回访记录、搭建供应商 |
| 产品资料 | 认证文件、培训材料、产品报价表 |
| 公司账号 | 阿里国际站、独立站、LinkedIn公司主页、企业邮箱、企微、Google Ads/Analytics、社媒企业号 |

### 处理
全部交接给接手人，不能带走。带走到新公司=违约。

## 🟡 混合资产（需判断边界，最容易出事）

### 判断标准
边界模糊的资产，通常涉及：
- 个人时间做但为公司业务服务
- 个人经验vs公司具体数据
- 公开渠道获取vs公司整理

### 关键判断问题
1. **是不是用公司设备/账号做的？** 是→公司可能主张
2. **是不是为公司的业务服务？** 是→公司可能主张
3. **数据来源是否公开？** 公开→归属偏向个人
4. **是否含商业敏感信息？** 是→不能带走

### 常见混合资产及处理
| 资产 | 判断 | 处理建议 |
|---|---|---|
| 行业联系人（KOL/客户） | 公开渠道找到的联系方式可重新建联；公司整理的带合作进度/价格的名单不能带走 | 方法论带走，具体数据留下。到新公司从公开渠道重新联系 |
| 合作方法论 | 个人经验 | 可以带走。但别用公司的具体案例、报价做新工作素材 |
| 自建工具/自动化 | 看是否公司时间/设备/为业务服务 | 技能本身带走；数据留一份给公司避免争议 |
| 供应商库 | 公开平台数据归属偏个人；用公司业务身份联系的公司可能主张 | 留给公司，方法论复用到其他品类 |

### 归属判断法律原则
下班后自己折腾的项目，法律上通常属于个人作品。但三个例外会让公司主张权利：
1. **用了公司设备/账号** — 公司电脑、公司飞书、公司账号联系供应商
2. **为公司的业务服务** — 即便下班后做，为业务做的项目归属有争议
3. **竞业限制范围** — 签了竞业风险更大，签保密相对小

## 🟢 个人资产（直接带走，无争议）

### 判断标准
- 纯个人项目，非公司业务
- 个人付费的工具/账号
- 个人学习资料、笔记

### 常见项目
| 项目 | 说明 |
|---|---|
| 个人技能（代码、流程、SOP） | 能力产物 |
| 个人付费账号（WorkBuddy等） | 直接带走，但清理公司相关数据 |
| 个人学习资料 | 语言学习、笔记 |
| 个人社媒账号 | 确认没绑公司邮箱 |

### 处理
直接带走。但注意：
- 个人付费账号要清理掉公司项目相关的memory/数据，避免后续混淆
- 个人社媒账号确认没绑定公司邮箱

## 风险等级判断

### 最高风险
- 同行业跳槽 + 保密协议
- 公司如果较真，最容易盯的是行业联系人资源

### 处理原则
1. **方法论带走，具体数据留下** — 怎么找联系人、怎么谈合作是能力；具体名单、价格、合同不能带
2. **重新建联而非复制** — 到新公司后从公开渠道重新联系
3. **混合资产留一份给公司** — 即便自己折腾的，留一份能避免归属争议
