# 离职交接7阶段流程

## 概览

```
0. 岗位选择 → 1. 信息收集 → 2. 资产盘点 → 3. 倒排时间表 → 4. 文档生成 → 5. 未闭环与对外通知 → 6. 最终确认与签收
```

每个阶段有明确的输入、动作、输出。

---

## 阶段0：岗位选择（优先执行）

### 目的
确定岗位类型，决定tab2工作交接看板的结构。不同岗位交接内容完全不同。

### 岗位类型与表格映射

| 岗位 | Tab2 包含表格 |
|---|---|
| **KOL/Influencer Marketing** | 工作交接总览、KOL合作详情、待跟进KOL、客户跟进、项目交接 |
| **社媒运营** | 工作交接总览、账号清单、内容日历、项目交接 |
| **外贸销售** | 工作交接总览、在跟客户、执行中订单、报价体系、项目交接 |
| **客服/Support** | 工作交接总览、未处理工单、FAQ与SOP、项目交接 |
| **产品经理** | 工作交接总览、需求清单、进行中项目、数据指标、项目交接 |
| **设计/内容** | 工作交接总览、进行中设计、素材库与版权、外包对接、项目交接 |
| **开发/技术** | 工作交接总览、代码仓库、线上服务、环境变量与配置、技术债与Bug、项目交接 |
| **其他** | 工作交接总览、自定义模块（按用户描述生成）、项目交接 |

### 输出
确定后的岗位类型和对应的表格定义，传递给阶段4文档生成。

---

## 阶段1：信息收集

### 目的
收集用户基本情况 + 接手人信息，决定后续5个阶段的输出基调。

### 收集方式
用AskUserQuestion批量问，分两组。不要一个个挤牙膏。

### 第一组：基本情况

| 问题 | 用途 |
|---|---|
| 离职日期 | 倒排时间表的基础 |
| 是否签过竞业/保密协议 | 决定资源能否带走，影响整个skill输出基调 |
| 下一份工作方向 | 判断哪些技能/资源可复用，哪些要主动切割 |
| 主要项目列表及归属 | 决定资产分类 |

### 第二组：接手人视角

| 问题 | 用途 |
|---|---|
| 谁接手你的工作 | 生成交接看板的核心 |
| 每块工作分别由谁接 | 填入工作交接看板 |
| 主管/最终确认人是谁 | 签收记录用 |
| 离职后是否允许短期联系 | 填入交接看板备注 |

**关键**：如果接手人还没定，先提醒用户确认再继续。接手人信息是整个交接看板的基础。

### 第三组：对外通知

| 问题 | 用途 |
|---|---|
| 哪些人需要通知离职（内部/外部合作方/KOL/客户/供应商） | 决定通知清单范围和邮件模板选择 |
| 分别用什么语言通知（中文/英文） | 决定生成哪些邮件模板 |
| 是否需要CC接手人 | 填入通知清单 |

### 输出
**不写入长期memory**。只在本次会话和生成的HTML/JSON文件中使用。离职交接信息敏感度极高，包含合同、客户、合作方、账号、付款等。

---

## 阶段2：资产盘点

### 目的
把所有要处理的东西分成三类，输出一张表。

### 分类方法
详见 [asset-classification.md](asset-classification.md)。

### 风险等级（非法律意见，必要时向HR/法务确认）
- **高风险**：建议交接并留痕，涉及商业敏感信息
- **中风险**：建议脱敏处理，只保留方法论，不复制具体数据
- **低风险**：可作为个人经验保留，但不要包含公司敏感信息

### 输出
- 🔴 公司资产清单（必须交接）
- 🟡 混合资产清单（需判断边界）
- 🟢 个人资产清单（直接带走）

### 关键决策点
- 签了保密/竞业 → 行业联系人、客户名单不能带走
- 同行业跳槽 → 风险最高，主动切割公司资源
- 自建项目 → 看是否公司时间/设备/为业务服务

## 阶段3：倒排时间表

### 目的
按离职日倒推，生成每天该做什么。

### 时间表模板（按周期调整）

#### 30天以上周期
- D-30：信息收集、资产盘点
- D-20：整理交接文档（工作看板、合作方、客户）
- D-15：账号权限移交
- D-10：项目状态文档、未闭环事项清单、与接手人过交接
- D-7：个人数据导出、准备对外通知
- D-3：设备清理、发送对外通知
- D-1：最终核对、逐模块签收、签字
- D-day：交设备、签确认书、拿离职证明

#### 12天短周期（参考）
- D-12：信息收集、资产盘点
- D-11：整理工作交接看板
- D-10：归档资料、整理未闭环事项
- D-9~D-8：账号权限移交、安全检查
- D-7：项目状态文档、过交接
- D-6：项目归属谈判
- D-5~D-4：个人数据导出、清理、准备对外通知
- D-3：设备清理、发送对外通知
- D-2：逐模块签收、核对
- D-1：签字、告别
- D-day：交设备、签确认书、拿离职证明

### 输出
交互式HTML倒排表，支持勾选、编辑、状态跟踪。

## 阶段4：文档生成

### 目的
按职责分模块生成交接文档，填充HTML工具箱。

### 占位符替换规则

| 占位符 | 替换为 |
|---|---|
| `{{USER_INFO}}` | 姓名：XX ｜ 岗位：XX ｜ 离职日期：XX ｜ 协议：XX ｜ 下份方向：XX |
| `{{USER_NAME}}` | 用户姓名 |
| `{{RESIGN_DATE}}` | 离职日期（YYYY-MM-DD） |
| `{{ROLE_TYPE}}` | 岗位类型 |
| `{{ROLE_TABLES}}` | **按阶段0岗位映射表生成tab2的全部table HTML**（替换模板中的默认表格） |
| `{{ROLE_CHECKLIST_ITEMS}}` | **AI动态生成7模块checklist JS对象** |
| `{{ROLE_SIGNOFF_MODULES}}` | 按岗位定制签收记录模块行 |
| `{{ROLE_SCHEDULE_ITEMS}}` | **AI根据阶段0-2所有信息动态生成倒排任务JS数组**。不预定义，每个岗位/每人生成不同的清单 |
| `{{ROLE_ACCOUNT_ROWS}}` | 权限移交表账号行HTML（按岗位生成） |
| `{{ROLE_SECURITY_ROWS}}` | 安全检查表账号行HTML |
| `{{ROLE_ACCOUNT_NOTE}}` | 移交方式说明 |
| `{{ROLE_SECURITY_NOTE}}` | 安全检查提示 |
| `{{ROLE_ACCOUNT_TYPES}}` | 公司账号类型描述 |
| `{{ASSET_YELLOW_ROWS}}` | 混合资产具体行 |
| `{{ASSET_GREEN_ROWS}}` | 个人资产具体行 |
| `{{RISK_REMINDER}}` | 风险提醒 |
| `{{EMAIL_TEMPLATES}}` | 按需生成的邮件模板 |

### 生成原则
- `{{ROLE_TABLES}}` 是tab2的全部table HTML，根据阶段0的岗位映射表生成
- 每个表格默认2-3个空行，用户在浏览器中可增删
- `{{ROLE_SCHEDULE_ITEMS}}` 由AI根据阶段0岗位+阶段1-2收集的所有信息动态推理生成。不预定义每种岗位的清单，每个用户得到不同的倒排任务

### 输出
所有内容内置在HTML工具箱的6个tab中，可直接在表格里填写，自动保存。

## 阶段5：未闭环事项与对外通知

### 目的
确保"看起来交接了，但还有未闭环事项"的问题被覆盖。同时处理好对外通知，确保沟通不断档。

### 5.1 未闭环事项

引导用户列出所有尚未闭环的事项：
- 等待回复
- 等待产品功能更新
- 待付款
- 待发布
- 待确认

每项填明：事项、当前状态、风险等级（高/中/低）、下一步动作、负责人、截止日期、相关链接、备注。

### 5.2 对外通知

1. 根据阶段1第三组回答，确认通知对象和语言需求。

2. 填写通知清单：合作方、关系类型、是否需要通知、通知方式、语言（中/英）、是否CC接手人、通知状态、备注。

3. 按场景选择邮件模板（4种，只生成用得到的）：
   - **对内中文** — 发给主管/HR/同事
   - **对外英文-正式** — 发给客户/供应商/B2B
   - **对外英文-KOL** — 发给KOL/influencer
   - **对外中文** — 发给国内合作方

用户可直接编辑模板中的方括号内容，然后复制使用。

### 输出
内置在HTML工具箱的"未闭环事项"和"对外通知"tab。

---

## 阶段6：最终确认与签收

### 目的
走之前逐项打勾，每块工作都有接收记录，全部✓才能安心离开。

### 6.1 七大模块Checklist
1. 工作交接
2. 账号权限
3. 项目归属
4. 个人数据
5. 设备清理
6. 保密协议遵守
7. 离职手续

### 6.2 逐模块签收记录
不是只在最后签个字，而是每块工作都有接收记录。模块名由 `{{ROLE_SIGNOFF_MODULES}}` 按岗位动态填充：
- 交接人
- 接手人
- 确认时间
- 确认方式（会议/书面/邮件/后台操作）
- 是否有遗留问题

### 6.3 交接风险评级
自动计算，显示在tab6顶部：
- 🟢 低风险：所有高优先级事项已交接，账号权限已移交
- 🟡 中风险：部分事项未确认接收，或存在未移交账号
- 🔴 高风险：存在未移交账号/未付款/未通知外部/无接手人

### 6.4 签字确认表
离职人、直属上级、HR。

### 输出
内置在HTML工具箱的"最终确认/签收"tab，可勾选，有进度条。

## HTML工具箱说明

### 单文件结构
所有内容在一个HTML文件里，6个tab：
1. 倒排表 — 交互式任务清单，支持新建分组/编辑/勾选，默认任务通用化
2. 工作交接看板 — 由 `{{ROLE_TABLES}}` 按岗位动态填充（总览 + 岗位特定表格 + 项目交接）
3. 资产与权限盘点 — 三类资产 + 权限移交清单 + 安全检查清单
4. 未闭环事项 — 可编辑表格，支持增删行
5. 对外通知 — 通知清单（含语言列） + 按需生成的邮件模板
6. 最终确认/签收 — 风险评级条 + checklist（含岗位特定项） + 逐模块签收（模块名按岗位）+ 签字确认

### 表格增删行
所有可编辑表格下方有"添加行"/"删除行"按钮。动态行数存入localStorage。

### 数据持久化
- localStorage（STORAGE_KEY: `resignation-toolkit-v3`），关浏览器再开还在
- 支持JSON导出备份/导入恢复
- **不写入长期memory**：离职信息敏感度极高，只在本次会话和HTML/JSON文件中使用

### 隐私与安全
- 默认不写入长期memory
- 只在本次会话和生成的HTML/JSON文件中使用
- 如果用户明确要求记住，再保存非敏感偏好
- HTML数据存在浏览器localStorage，不上传任何服务器
