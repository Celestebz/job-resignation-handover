# 离职交接5阶段流程

## 概览

```
1. 信息收集 → 2. 资产盘点 → 3. 倒排时间表 → 4. 文档生成 → 5. 最终核对
```

每个阶段有明确的输入、动作、输出。

---

## 阶段1：信息收集

### 目的
收集用户基本情况，决定后续4个阶段的输出基调。

### 收集方式
用AskUserQuestion批量问，不要一个个挤牙膏。一次问4个核心问题。

### 核心问题

| 问题 | 用途 |
|---|---|
| 离职日期 | 倒排时间表的基础 |
| 是否签过竞业/保密协议 | 决定资源能否带走，影响整个skill输出基调 |
| 下一份工作方向 | 判断哪些技能/资源可复用，哪些要主动切割 |
| 主要项目列表及归属 | 决定资产分类 |

### 可选问题（根据情况追加）
- WorkBuddy/各种SaaS账号是公司开通还是个人付费
- 自建工具/自动化是公司时间还是下班后做的
- 是否用了公司设备/账号

### 输出
存到memory，避免重复问。

---

## 阶段2：资产盘点

### 目的
把所有要处理的东西分成三类，输出一张表。

### 分类方法
详见 [asset-classification.md](asset-classification.md)。

### 输出
- 🔴 公司资产清单（必须交接）
- 🟡 混合资产清单（需判断边界）
- 🟢 个人资产清单（直接带走）

### 关键决策点
- 签了保密/竞业 → 行业联系人、客户名单不能带走
- 同行业跳槽 → 风险最高，主动切割公司资源
- 自建项目 → 看是否公司时间/设备/为业务服务

---

## 阶段3：倒排时间表

### 目的
按离职日倒推，生成每天该做什么。

### 时间表模板（按周期调整）

#### 30天以上周期
- D-30：信息收集、资产盘点
- D-20：整理交接文档（合作方、客户、报价）
- D-15：账号权限移交
- D-10：项目状态文档、与接手人过交接
- D-7：个人数据导出
- D-3：设备清理
- D-1：最终核对、签字
- D-day：交设备、签确认书、拿离职证明

#### 12天短周期（参考）
- D-12：信息收集、资产盘点
- D-11：整理交接文档
- D-10：归档资料
- D-9~D-8：账号权限移交
- D-7：项目状态文档、过交接
- D-6：项目归属谈判
- D-5~D-4：个人数据导出、清理
- D-3：设备清理
- D-2：确认交接、核对
- D-1：签字、告别
- D-day：交设备、签确认书、拿离职证明

### 输出
交互式HTML倒排表，支持勾选、编辑、状态跟踪。

---

## 阶段4：文档生成

### 目的
按职责分模块生成交接文档模板。

### 四个模板
1. **合作方交接** — 联系人、合同状态、付款进度、坑点提醒
2. **客户跟进交接** — 询盘状态、谈判进度、下一步动作
3. **项目交接** — 背景、当前进度、下一步、负责人
4. **账号权限清单** — 平台、账号、权限、移交对象

### 输出
所有模板内置在HTML工具箱的"交接模板"tab，可直接在表格里填写，自动保存。

---

## 阶段5：最终核对

### 目的
走之前逐项打勾，全部✓才能安心离开。

### 七大模块
1. 工作交接
2. 账号权限
3. 项目归属
4. 个人数据
5. 设备清理
6. 保密协议遵守
7. 离职手续

### 输出
内置在HTML工具箱的"最终核对"tab，可勾选，有进度条。

---

## HTML工具箱说明

### 单文件结构
所有内容在一个HTML文件里，4个tab：
1. 倒排表 — 交互式任务清单
2. 资产盘点 — 可编辑表格
3. 交接模板 — 可填写表格
4. 最终核对 — 可勾选checklist

### 占位符说明
模板中用`{{占位符}}`标记需要根据用户情况替换的内容：
- `{{USER_INFO}}` — 资产盘点tab顶部的用户信息
- `{{USER_NAME}}` — 用户姓名（签字表、账号表）
- `{{RESIGN_DATE}}` — 离职日期
- `{{ASSET_YELLOW_ROWS}}` — 混合资产具体行
- `{{ASSET_GREEN_ROWS}}` — 个人资产具体行
- `{{RISK_REMINDER}}` — 风险提醒
- `{{PROJECT_NAMES}}` — 项目名称列表
- `{{PROJECT_OWNERSHIP_NOTE}}` — 项目归属处理说明
- `{{COMPANY_ASSET_HANDOVER}}` — 公司资产交接任务
- `{{MIXED_ASSET_HANDOVER}}` — 混合资产处理任务
- `{{PERSONAL_DATA_NOTE}}` — 个人数据导出说明

### 数据持久化
- localStorage，关浏览器再开还在
- 支持JSON导出备份/导入恢复
- STORAGE_KEY: `resignation-toolkit-v1`
