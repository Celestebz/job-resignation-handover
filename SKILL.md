# 离职交接

5阶段闭环：信息收集 → 资产盘点 → 倒排时间表 → 文档生成 → 最终核对。

## 核心流程

### 阶段1：信息收集

用AskUserQuestion批量问4个核心问题，不要一个个挤：

| 问题 | 选项 | 用途 |
|---|---|---|
| 竞业/保密协议 | 都没签/签过保密/签过竞业/都签了 | 决定资源能否带走 |
| 下份方向 | 同行业/换行业/休息/不透露 | 判断资源可复用性 |
| 自建项目归属 | 公司派做/自己折腾/混合/需谈 | 决定数据归属 |
| SaaS账号性质 | 公司开通/自己付费/不确定 | 决定走后能否继续用 |

还要问**离职日期**，这是倒排表的基础。

### 阶段2：资产盘点

把所有资产分三类，详见 [references/asset-classification.md](references/asset-classification.md)。

- 🔴 **公司资产** — 必须交接，不能带走
- 🟡 **混合资产** — 需判断边界，最容易出事
- 🟢 **个人资产** — 直接带走，无争议

**关键判断**：签了保密/竞业 + 同行业跳槽 = 风险最高，行业联系人资源最容易出问题。

### 阶段3-5：生成HTML工具箱

复制 `assets/离职交接模板.html` 到用户工作目录，根据阶段1-2的收集结果替换占位符。

**占位符替换规则**：

| 占位符 | 替换为 |
|---|---|
| `{{USER_INFO}}` | 姓名：XX ｜ 岗位：XX ｜ 离职日期：XX ｜ 协议：XX ｜ 下份方向：XX |
| `{{USER_NAME}}` | 用户姓名 |
| `{{RESIGN_DATE}}` | 离职日期（YYYY-MM-DD） |
| `{{ASSET_YELLOW_ROWS}}` | 混合资产具体行（根据用户项目生成） |
| `{{ASSET_GREEN_ROWS}}` | 个人资产具体行 |
| `{{RISK_REMINDER}}` | 基于用户情况的风险提醒 |
| `{{PROJECT_NAMES}}` | 项目名称列表 |
| `{{PROJECT_OWNERSHIP_NOTE}}` | 项目归属处理说明 |
| `{{COMPANY_ASSET_HANDOVER}}` | 公司资产交接任务名 |
| `{{MIXED_ASSET_HANDOVER}}` | 混合资产处理任务名 |
| `{{PERSONAL_DATA_NOTE}}` | 个人数据导出说明 |

**倒排表日期**：根据离职日期计算D-30到D-day（或短周期D-12到D-day）。如果离职日期距今不足15天，用12天短周期模板；否则用30天标准模板。

**资产盘点表填充**：根据用户回答的具体项目，生成混合资产和个人资产的表格行。每行包含：项目名、判断、风险点、处理建议。

## HTML工具箱结构

单文件4个tab：

1. **倒排表** — 交互式任务清单，支持勾选/编辑/添加，状态高亮（今日/逾期/完成），顶部统计+进度条
2. **资产盘点** — 红/黄/绿三类表格，单元格可点击编辑，自动保存
3. **交接模板** — 4个模板（合作方/客户/项目/账号），可直接填写
4. **最终核对** — 7大模块checklist，勾选有进度条

数据存localStorage，支持JSON导出备份/导入恢复。

## 关键设计决策

1. **HTML而非md** — 倒排表是跨几周的todo，需要状态跟踪和勾选，静态md做不到
2. **单文件** — 4个tab在一个HTML里，方便使用和分享
3. **资产分类是核心** — 多数纠纷出在"这东西到底算谁的"，要主动逼用户想清楚
4. **竞业/保密前置问** — 决定整个skill输出基调
5. **个性化填充** — 模板用占位符，根据用户情况动态生成

## 注意事项

- 倒排表任务要根据用户实际岗位定制（外贸/KOL/开发等不同岗位交接内容不同）
- 混合资产的风险提醒要具体，不要泛泛而谈
- 如果用户签了竞业，要强烈提醒行业联系人不能带走
- 走之前提醒用户清理个人付费账号中的公司相关数据
